# DE-KEA
_IKEA? I hardly know her!_

Get rid of the IKEA in your life. DE-KEA.

## User journey
- On the homepage you see a demo video start playing immediately.
  - There's a two-line explanation of the product and a CTA.
- Click the 'Take photo' button
- Take a photo of a room (we're encouraged to do it in landscape).
- Now we go to a half and half page layout.
  - The top half has the current image, the bottom half is the conversation.
  - Each pane can be expanded to full page or re-
  - Above the top half is a thin band with 'DE-KEA: Interior design without IKEA.'
- A new image is returned to the image pane. All the IKEA objects hav been removed from it.
- The conversation pane includes the returned image from the agent, followed by rich rendered Markdown which lists the identified IKEA objects along with witheringly critical comments about each one.
  - Note that the recycling centre line would need web browsing.
```markdown
## Removed:
1. A tacky red lamp
2. Four astonishingly gaudy prints on the right hand wall
3. A BILLY bookcase (yawn)
<!--v1.4 Pi required for the web search/browsing subagent. We'll hardcode it in for v1.1 but remove for v1.2 and v1.3.-->
What's your postcode? I'll find your nearest recycling centre.

Then we can start designing a room that is tasteful, a room that you're proud of.
```
- The user responds. They say they'd also like to replace the sofa. Another image is returned in the conversation, this time without the sofa as well.
- We now go through the items. They're asked what style of lamp they'd like -- modern, retro, classic? They can try a few options. They see how a new object would look in the room.
- (v1.4) Each time the user is happy with a couple of options for the room, they are told where they can get the new object from. For example there's a British Heart Foundation down the road with good quality sofas.
  - These options should typically be:
    - eBay, Gumtree, Facebook marketplace
    - Freecycle
    - Charity shops
    - Street markets
    - Antique shops
    - Auction houses

## Technical

- Pi may be required to wrap the other agents as sub-agents (e.g. for research).
- Use ElevenAgents (https://elevenlabs.io/agents) wherever needed.
- We can use OpenRouter for direct LLM API access.
- This is a SvelteKit app ideally deployed Cloudflare Pages, if that's possible with the Pi agent. 
  - If Cloudflare, manage the deployment through the CLI.
  - If it's not possible to run Pi on Cloudflare Pages then we'll need to just use ElevenAgents and give it lots of tools.
